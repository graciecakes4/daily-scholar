# 2026-06-29

Editorial UI + production stability release. v2.3 shipped the scope library; **v2.4 rebuilds the navigation around it** and folds in the prod-login + mobile bugs that surfaced once the multi-tenant beta was live. The sticky top bar is replaced by a 280px persistent editorial left sidebar (cream paper + Fraunces serif + warm gold accent) reorganized into hierarchical groups (Read · Scope · Account · Help) with a pinned active-scope chip at the top of the rail. Mobile keeps the bottom tab bar but reskinned to match, with the More sheet regrouped to mirror the sidebar IA. A new "Replay tutorials" picker on both surfaces re-fires a specific product tour via the existing tour-state plumbing. Plus three production-impacting fixes: same-origin `/api/*` proxy unblocking prod login after the Cloudflare Access removal (hotfix #46, already on main), iOS Dynamic Island safe-area + post-onboarding redirect loop + uvicorn keep-alive timeout (#47), and a per-tour reset endpoint to power the new picker (#48). **Zero new migrations, zero new dependencies, one new Railway env var (`BACKEND_INTERNAL_URL`), one removed (`NEXT_PUBLIC_API_URL`), one opt-in backend env var (`COOKIE_DOMAIN`) added during the hotfix debugging and immediately rendered moot by the same-origin collapse.** Every existing page, route, and guard still works — nav-only structural change.

## Highlights

### Editorial theme foundation (PR #48)

Tailwind palette extended with `paper / ink / muted / rule / gold / gold-dark / rust / moss` tokens; legacy `scholar / surface` tokens preserved so existing pages keep rendering unchanged. `font-serif` now maps to Fraunces, `font-mono` to IBM Plex Mono. `globals.css` imports the three Google Fonts (Fraunces + IBM Plex Mono + Inter), defines CSS custom properties for the full palette, sets the warm-paper body background with two radial glows and an SVG-noise `body::before` overlay, and repoints scrollbar thumbs to the warm tones. Locking the design tokens in first means the rest of the diff is pure consumption of those tokens — easier to review, easier to revert in isolation if needed.

### Editorial left sidebar + active-scope chip (PR #48)

New `Sidebar.tsx` (280px persistent rail on `md+`) with wordmark, pinned `ActiveScopeChip`, four named groups (Read / Scope / Account / Help) with hierarchical sub-items, and a `UserChip` footer with a small popover preserving the existing logout. Hides on `/login`, `/signup`, `/account/*` so the auth shell still owns the viewport on those routes.

New `ActiveScopeChip.tsx` calls `getActiveScope()` on mount and renders three states: loading skeleton (height-stable), empty prompt linking to `/scopes/picker`, and ready chip showing scope name + topic count + `edited Xh ago` (relative time from `scope.updated_at` via an inline `formatRelative` helper).

`app/layout.tsx` rewritten as a two-column flex shell; `themeColor` set to `#F2EBDD` so iOS chrome blends into the paper, and the main column carries `paddingTop: calc(env(safe-area-inset-top) + 2rem)` so the Dynamic Island clearance from the bugfix PR is preserved.

Flat nav had grown to 8+ top-level items and Settings was conflating account, scope, and notifications — the sidebar gives every IA group room to breathe, makes the active scope a persistent first-class affordance (one click to switch), and the warm paper aesthetic better matches a "daily reading" product.

### Mobile reskin + IA parity (PR #48)

`MobileTabBar.tsx` reskinned to the cream/gold palette: gold underline for active tab, paper-2 sheet background, Fraunces section labels. Bottom tabs unchanged at Today / Papers / Topics / Quizzes / More. More sheet regrouped into the same Scope / Account / Help sections as the sidebar, with matching sub-items and icons. Mobile and desktop now feel like the same product; mirroring the sidebar's IA in the More sheet means muscle memory carries across.

### Tour replay picker (PR #48)

Help group on both surfaces now has a "Replay tutorials" entry. Tapping it swaps the same section in-place into a small picker with three choices — Dashboard / Scope library / Topics. Selecting one calls the new `resetTour(id)`, refreshes `/auth/me`, then `router.push`-es to the tour's `fire_on_path`. A pulsing gold play icon plus an IBM Plex Mono "loading" tag covers the in-flight state; back chevron returns to the default Help list.

Backend: `PUT /auth/tour-reset` now accepts an optional `?tour_id=<id>` query param — with it, only that key's seen-version zeroes (rest of `tour_state` untouched); without it, existing full-reset behavior. Validated against `KNOWN_TOUR_IDS`; unknown ids return 400 with the same error shape as `mark_tour_completed`. Two new tests in `backend/tests/test_tour_versioning.py` cover per-tour reset happy path + unknown-id 400.

Per-tour reset (vs. full reset) lets a user re-run just the walkthrough they want without nuking unrelated tour progress.

### iOS Dynamic Island safe-area spacing (PR #47)

Added `paddingTop: env(safe-area-inset-top)` to the sticky `<nav>` in the root layout. `viewportFit: "cover"` was already set so the page extends under the Dynamic Island; without the inset, the nav contents sat behind the island on iPhone 14 Pro and later. The sidebar overhaul in PR #48 preserves this via `calc(env(safe-area-inset-top) + 2rem)` on the main column padding.

### Post-onboarding redirect loop (PR #47)

`skipOnboarding()` and `completeOnboarding()` now emit the `AUTH_CHANGED_EVENT` window event after success. Also extended `OnboardingGuard`'s `SKIP_PREFIXES` with `/scopes` and `/settings/scope`.

`useAuth` is a per-component hook with no React context — cross-instance sync only happens via `AUTH_CHANGED_EVENT`. `login()` / `logout()` emitted it, but the onboarding mutations didn't. The layout-mounted `OnboardingGuard` kept a stale `user.onboarded === false`, bounced users back to `/onboarding`, and looped until something else triggered a refetch. The `SKIP_PREFIXES` additions are defense-in-depth so the guard can't yank users off scope-setup surfaces mid-flow if onboarded state ever goes briefly stale again.

### uvicorn keep-alive (PR #47 — proxy `ECONNRESET` fix)

Added `--timeout-keep-alive 75` to the uvicorn invocation in both the production Dockerfile CMD and `start.sh`. Next.js 16's `rewrites()` proxy (undici-backed) pools outgoing connections with ~75s idle. uvicorn's default `--timeout-keep-alive` is 5s, so the proxy reused already-closed sockets and threw `socket hang up` / `ECONNRESET` on `/auth/me`, `/user/active-scope`, `/scopes/mine`, `/daily`, etc. Requires a rebuild — the flag is baked into the CMD.

### Mobile More sheet polish (PR #47)

Added a Notifications row to the More bottom sheet (matching the new desktop nav entry); tightened the Settings active-state highlighting so it no longer falsely activates for non-scope `/settings/*` paths. Subsequently superseded by the IA regroup in PR #48, but documented for the audit trail.

## Hotfixes folded in from main

### Same-origin proxy for `/api/*` (hotfix #46)

Production login was broken after deleting the Cloudflare Access apps on `scholar.daily-scholar.com` + `api.daily-scholar.com`. CF Access had been masking a real architectural issue: with the frontend and backend on separate subdomains, the CSRF middleware's double-submit-cookie pattern can't work because the `ds_csrf` cookie set by `api.daily-scholar.com` isn't readable by JS at `scholar.daily-scholar.com`. Login never POSTed in prod before because CF Access was authenticating users at the edge and stamping `Cf-Access-Authenticated-User-Email` on every request.

Collapsed the architecture to **same-origin** via a Next.js rewrite: `frontend/next.config.js` adds `rewrites()` that proxies `/api/:path*` → `${BACKEND_INTERNAL_URL}/:path*`, `frontend/lib/api.ts` `API_BASE` defaults to `/api` so all browser calls go through the proxy, and `frontend/hooks/useWebPush.ts` drops its own `API_BASE` copy and imports the shared one. Cookies are scoped to `scholar.daily-scholar.com`, JS can read them, the cross-subdomain CSRF + CORS class of bugs goes away. No backend code change required — cookies follow the URL the browser hit, not the proxy destination.

Cherry-picked back into develop (commit `e2abd80` "Cross-subdomain cookie domain + middleware reorder for prod login") so this release PR doesn't conflict. That same commit also (a) introduced an opt-in `COOKIE_DOMAIN` env var read by `_cookie_domain()` in `backend/api/auth.py` and `backend/middleware/csrf.py` — unset means origin-scoped cookies, which is the desired behavior under same-origin, so prod should leave it unset; left in place as a no-op safety valve — and (b) reordered the `backend/main.py` middleware stack so CORS is the OUTERMOST layer (last `add_middleware` call wins under Starlette's LIFO wrapping). Request flow is now `CORS → RateLimit → CSRF → handler`, response flow `handler → CSRF → RateLimit → CORS`, so CORS headers attach to every response — including early 403s from the inner CSRF / rate-limit middlewares. Otherwise an inner early-return surfaces in the browser as a misleading "CORS error" because the response wouldn't carry CORS headers.

## Migration shape

**Zero new migrations.** The only schema-touching code in the cycle is the per-tour reset endpoint, which reads/writes the existing `users.tour_state` JSON column added in v2.2's `0011_tour_version_seen`.

## Notes

- **No new Python deps, no new npm deps.** Net new dependency surface is empty for v2.4.
- **One new Railway env var, one removed.** Frontend service needs `BACKEND_INTERNAL_URL=http://backend.railway.internal:8000` set BEFORE the v2.4 image will build correctly (Next.js bakes `rewrites()` destinations into the route manifest at build time). `NEXT_PUBLIC_API_URL` should be unset — leaving it would override the `/api` default and bypass the proxy. The backend service's `api.daily-scholar.com` custom domain can stay one release as a safety net; delete after v2.4 is verified.
- **Production Dockerfile rebuild required** — the uvicorn `--timeout-keep-alive 75` flag is baked into the CMD; no live-reload path will pick it up.
- **Cookies are now first-party.** `ds_session` + `ds_csrf` set by `scholar.daily-scholar.com` (not the deleted `api.*` host). Old cookies from the cross-subdomain era expire naturally; first login on the new build issues fresh ones.
- **CORS allowlist becomes dead code.** `FRONTEND_URL` + `_resolve_cors_origins()` in `backend/main.py` are no-ops under same-origin; harmless to leave; can be removed in v2.5 alongside the OAuth work.
- **Visual mockups gitignored.** Three nav layout options were prototyped in `nav-mockups/` as standalone HTML before implementation; `.gitignore` now matches `*mockups/` so the folder stays local. Mirrors the v2.1 pattern for `daily scholar/`.
- **`UserMenu.tsx` left on disk** (unused by the new layout) for any future use; safe to delete in a later cleanup PR if it stays orphaned.
- **`TOUR_CHOICES` duplicated** between `Sidebar.tsx` and `MobileTabBar.tsx` with a comment pointing at backend `KNOWN_TOUR_IDS` as the source of truth — if a fourth tour is added, both arrays need updating. Could be DRY'd by exposing tour metadata via an endpoint or a shared `lib/tours.ts`; not worth it for three entries.
- **Tests.** Backend `test_tour_versioning.py` 14 / 14 pass (12 existing + 2 new). Frontend `tsc --noEmit` clean. Manual: login → onboarding → dashboard → switch scope from sidebar chip → replay each tour from picker, both desktop and mobile breakpoints.
- **Not yet verified.** PWA install flow on the new layout (no functional change but worth a smoke check), iOS Safari with `viewportFit: cover` on a physical device with Dynamic Island.
